# 변경 이력

## 1.6.1

- 멀티 모니터 환경에서 그리드가 화면 위쪽 일부를 비워 두고(맨 윗줄 제외) 아래로 밀려 그려지던 문제 수정 — 보조 모니터가 주 모니터보다 아래로 내려간 배치에서 좌표 변환이 어긋나, 그리드 표시·셀 선택·창 스냅 위치가 모두 그만큼 밀리던 버그. 가장자리 절반 스냅·창 크기 고정의 위치 보정에도 동일하게 적용.

## 1.6.0

- 트랙패드 단축키를 **"드래그 중 입력"** 방식으로 변경: 창을 드래그하는 도중 단축키(기본 ⌃⌥G)를 누르면 우클릭과 똑같이 그리드가 켜지고, 셀을 고른 뒤 손을 떼면 배치됨. (1.5.0의 클릭-커밋 단독 모드는 제거)
- 단축키를 **설정에서 변경** 가능: 설정… → 그리드 단축키에서 원하는 조합으로 녹화.
- macOS **시스템·앱 기본 단축키는 지정 불가**: ⌘Space·⌃↑·⇧⌘4·⌘C 등 충돌 위험 단축키를 처음부터 막고, 수정키 없는 키도 거부.

## 1.5.0

- 트랙패드용 그리드 단축키 추가: **⌃⌥G** 를 누르면 커서 아래 창으로 그리드 모드가 켜지고(드래그·우클릭 불필요), 이동해 셀을 고른 뒤 클릭하거나 ⌃⌥G 를 다시 누르면 배치됨(Esc 취소). 기존 "드래그 중 우클릭" 방식은 그대로 유지.

## 1.4.1

- 설정의 그리드 열·행에 **[적용] 버튼** 추가 — 숫자를 고친 뒤 Enter를 누르지 않아도 적용됨(Enter로도 적용 가능).
- 창 크기 고정 안내 HUD의 글자 대비 개선 — 반투명 재질을 불투명 진한 배경으로 바꿔 밝은 배경·화면 녹화에서도 또렷하게 보임.

## 1.4.0

- 창 크기 고정 대상 미리보기: 비율을 고른 뒤 마우스를 움직이면 커서 아래 창이 강조 표시되어, 어떤 창이 바뀔지 클릭 전에 보입니다.

## 1.3.0

- 창 크기 고정 추가: 메뉴바 → **창 크기 고정**에서 비율(4:3 720×480, 16:9 1280×720 / 1920×1080)을 고른 뒤 바꿀 창을 클릭하면 그 크기로 맞춰짐. 좌상단 위치는 유지하고 화면 가시 영역 안으로 보정. (Esc·우클릭으로 취소)

## 1.2.0

- 앱별 예외 목록 추가: 지정한 앱(게임 등)이 맨 앞일 때는 그리드·가장자리 스냅이 마우스 입력에 일절 개입하지 않음 — 드래그·우클릭을 강제하는 게임에서 입력이 가로채이던 문제 해결. 설정 창에서 앱을 추가/제거.

## 1.1.2

- 드래그 감지 때문에 다른 앱 클릭/입력이 지연되던 문제 수정 (창 조회를 첫 드래그 시점에 비동기로 처리, 순수 클릭에는 영향 없음)

## 1.1.1

- 그리드 스냅이 Dock·상단 메뉴바 영역을 침범하던 문제 수정 (사용 영역 기준으로 셀 계산)

## 1.1.0

- 가장자리 절반 스냅 추가: 창을 그냥 끌어서 화면 **좌·우·아래** 끝에 대면 절반, **위** 끝에 대면 최대화
- 텍스트 선택·리사이즈와 구분(창이 실제로 이동할 때만 발동) · 메뉴바/Dock 영역은 침범하지 않음
- 설정에서 "가장자리 절반 스냅" 켜기/끄기 가능 (기본 켜짐)

## 1.0.2

- 새 앱 아이콘 적용 (그리드 + 캐릭터 로고)

## 1.0.1

- 동작 방식 변경: 오른쪽 버튼을 계속 누르지 않고, 드래그 도중 **오른쪽 클릭 한 번**으로 그리드를 켭니다
- 왼쪽 버튼을 놓으면 스냅 완료 · 다시 오른쪽 클릭하거나 Esc로 취소

## 1.0.0

- 첫 릴리스 🎉
- 창을 드래그하는 도중 우클릭하면 화면에 그리드 가이드 표시
- 셀 블록을 끌어 놓으면 창이 해당 영역으로 스냅(이동·리사이즈)
- 그리드 열·행 수 설정(기본 6×4)
- 메뉴바 상주 · Sparkle 자동 업데이트
